第二章まとめ：式・if文・switch文・ブロック

1. 式と文
- 式：評価すると必ず「値」が得られる記述。
	- 例：`a + b`（計算結果の値）、`x > y`（true/false の値）
- 文：プログラムの「処理・効果」を表す単位で、式を含むこともある。
	- 例：`a = b;`（代入という動作）、`cout << x;`（表示という動作）
- 「式には値があり、文には効果がある」という整理を学んだ。

2. bool型と真偽の扱い
- C++ の真偽は `bool` 型で表し、値は `true` / `false`。
- 整数値を条件式として評価すると、0 は false、それ以外は true とみなされる。
- `int` は多くの整数値を表すのに対し、`bool` は真か偽の2値のみを持つ。

3. 条件分岐 if 文
- `if (条件)` で条件が真のときだけ処理を行う。
	- 例：入力した整数が正のとき「その値は正です。」と表示するプログラム。
- `if (条件) … else …` で条件が偽のときの処理も書ける。
	- 例：正なら「正です」、それ以外なら「0か負です」と表示するプログラム。
- `==` は「等しいかどうかを比較」する演算子であり、`=` は「代入」を行う演算子である。
- 入力された2つの整数が等しいかどうかを、`if (a == b)` で判定するプログラムを書いた。

4. 論理演算子と短絡評価
- `&&`（かつ）、`||`（または）を使って条件を組み合わせられる。
- 「短絡評価（ショートサーキット）」：
	- `&&` では左側が false のとき、右側は評価されない。
	- `||` では左側が true のとき、右側は評価されない。
- これにより、「0 除算」など危険な計算を避ける書き方を学んだ。
	- 例：`if (x != 0 && 10 / x > 1)` のとき、`x != 0` が false なら `10 / x` は実行されない。

5. switch 文
- 同じ変数の値に応じて分岐する場合に `switch` 文を使える。
	- 例：`switch (n) { case 1: …; case 2: …; default: …; }`
- `case` ラベルごとに処理を書き、最後に `break;` を置かないと次の `case` へ実行が続く（フォールスルー）。
- `default` はどの `case` にも当てはまらないときに実行される。
- `switch` の条件式に使えるのは整数型や列挙型などであり、文字列や浮動小数点は使えないことも学んだ。
- 点数 `score` を 10 で割った値で場合分けして、A/B/C/F を表示するパターンを学んだ。

6. ブロックとスコープ
- `{ }` で囲まれた部分を「ブロック」と呼び、その中で宣言した変数はそのブロックの中でだけ有効（スコープ）である。
- ブロックの外からブロック内の変数を参照すると、コンパイルエラーになる。
	- 例：`if (true) { int x = 10; } cout << x;` は `x` がスコープ外のためエラー。
- 必要な変数をどの範囲で使うかを意識して、宣言場所を決める重要性を学んだ。

7. 条件演算子（?:）
- `条件 ? 式1 : 式2` という形で、1行で値を選ぶ書き方ができる。
	- 例：`int z = (x > y) ? x : y;` は x と y の大きい方を z に代入する。
- 整数 `a` が偶数なら "even"、奇数なら "odd" を出力するコード：
	- `cout << ((a % 2 == 0) ? "even" : "odd");`

8. 総合問題で身につけたこと
- 2つの整数を昇順に並べ替えるとき、`if (a > b)` で大小を判定し、一時変数を使って値を入れ替える。
	- 例：`int t = a; a = b; b = t;`
- ネストした `if` 文では、インデントと `else` がどの `if` に対応しているかに注意する必要がある。
- 点数に応じて成績を表示するなど、「条件に応じて処理を変える」プログラムを自分で組み立てる練習をした。

第二章では、C++ における「条件分岐の基本（if / switch）」「式と文の違い」「bool と評価結果」「ブロックとスコープ」など、今後のプログラム全体の土台になる考え方を一通り学んだ。
